## 🎯 작성 원칙

### 핵심 원칙

1. **학습자 중심**: 초중급 학습자의 눈높이에 맞춰 작성
2. **실습 중심**: 이론보다는 실제로 해보는 것에 중점
3. **단계적 접근**: 쉬운 것부터 어려운 것으로 점진적 난이도 상승
4. **명확한 설명**: 전문 용어 사용 시 반드시 설명 추가
5. **검증된 코드**: 모든 코드는 실제 헥사보드에서 테스트 완료

### 문체 및 톤

- **존댓말 사용**: "~합니다", "~해보세요"
- **친근한 톤**: 대화하듯 부드럽게
- **명확한 표현**: 모호한 표현 지양
- **격려와 지원**: 학습자를 격려하는 표현 사용

**좋은 예시**:

> "버튼 입력을 읽는 것은 매우 간단합니다. 한 번 시도해볼까요?"

**나쁜 예시**:

> "버튼 입력 감지는 디지털 신호 처리의 기초이며..."

## 📖 섹션별 작성 가이드

### 1. 학습 목표 (📚 이 챕터에서 배울 내용)

**작성 요령**:

- 구체적이고 측정 가능한 목표 3~5개
- "~할 수 있다" 형식으로 작성
- 체크박스 형식으로 제공

**예시**:

```markdown
- [ ] 버튼의 Pull-up/Pull-down 저항 개념을 이해할 수 있다
- [ ] GPIO 핀으로 버튼 입력을 읽을 수 있다
- [ ] 버튼 A와 B를 동시에 제어할 수 있다
```

### 2. 이론 설명 (📖)

**작성 요령**:

- 꼭 필요한 이론만 간결하게
- 복잡한 개념은 비유나 예시 활용
- 다이어그램이나 그림으로 보조 설명
- 핵심 포인트를 별도로 강조

**구조**:

```markdown
### [주제]: [제목]

#### 개념 설명

[2-3 문단으로 개념 설명]

**핵심 포인트**:

- 포인트 1
- 포인트 2

#### 동작 원리

[동작 원리 설명 + 간단한 코드 예시]
```

**분량 가이드**:

- 이론 중심 챕터(PART 1): 50% 이론, 50% 실습
- 실습 중심 챕터(PART 2~7): 30% 이론, 70% 실습

### 3. 실습 준비 (🔧)

**작성 요령**:

- 필요한 모든 하드웨어/소프트웨어 명시
- 회로 구성이 필요한 경우 명확한 연결도 제공
- 주의사항 반드시 포함

**회로도 제공 시**:

```markdown
**연결 방법**:
```

## 헥사보드 센서

GPIO 32 <--> DATA
GND <--> GND
3.3V <--> VCC

```

**주의사항**:
⚠️ 전원 연결 전 연결을 다시 한번 확인하세요!
```

### 4. 실습 코드 (💻)

**작성 요령**:

- 단계별로 나누어 설명 (1단계 → 2단계 → 3단계)
- 각 단계마다 코드 + 설명 제공
- 중요한 코드 라인은 별도 설명 추가
- 전체 코드와 부분 코드를 적절히 혼합

**코드 구조**:

```python
# [파일명: chXX_example.py]
# [전체적인 코드 설명 1-2줄]

# 필요한 라이브러리 임포트
from machine import Pin
import time

# 설정 값 정의
BUTTON_A_PIN = 35
BUTTON_B_PIN = 34

# 메인 로직
def main():
    # 버튼 초기화
    button_a = Pin(BUTTON_A_PIN, Pin.IN, Pin.PULL_UP)

    # 메인 루프
    while True:
        # 버튼 상태 읽기
        if button_a.value() == 0:  # 버튼 눌림
            print("Button A pressed!")
        time.sleep(0.1)

# 실행
if __name__ == "__main__":
    main()
```

**코드 설명 작성**:

```markdown
**코드 설명**:

- **Line 4-5**: 버튼 핀 번호를 상수로 정의합니다
- **Line 9**: Pull-up 저항을 사용하도록 설정합니다
- **Line 13**: 버튼이 눌리면 0을 반환합니다 (Pull-up 방식)
```

### 5. 동작 확인 (✅)

**작성 요령**:

- 정상 동작 시 예상 결과 명시
- 실행 방법을 단계별로 설명
- 스크린샷이나 동작 영상 캡처 포함

**예시**:

```markdown
### 예상 결과

**정상 동작 시**:

- ✅ 버튼 A를 누르면 "Button A pressed!" 출력
- ✅ 버튼 B를 누르면 "Button B pressed!" 출력
- ✅ 버튼을 떼면 출력이 멈춤

**동작 화면**:
![실행 결과](../images/chapter04/result.png)
```

### 6. 문제 해결 (🐛)

**작성 요령**:

- 실제로 자주 발생하는 문제 3~5개 선정
- 증상 → 원인 → 해결 방법 순서로 작성
- FAQ 형식으로 추가 질문/답변 제공

**구조**:

```markdown
### 문제 1: [문제 제목]

**증상**:

- [구체적인 증상]

**원인**:

- [원인 분석]

**해결 방법**:

1. [단계별 해결 방법]
2. [단계별 해결 방법]
```

### 7. 도전 과제 (🚀)

**작성 요령**:

- 난이도별로 3~4개 과제 제공
  - 기본 과제 ⭐️: 챕터 내용 복습
  - 심화 과제 ⭐️⭐️: 응용 및 확장
  - 창의 과제 ⭐️⭐️⭐️: 자유 창작
- 힌트 제공 (바로 답은 주지 않음)
- 예상 소요 시간 명시

**예시**:

```markdown
### 기본 과제 ⭐️

**과제 1**: 버튼 카운터 만들기

- **목표**: 버튼을 누를 때마다 카운트가 증가하는 프로그램 작성
- **힌트**: 변수를 사용하여 누른 횟수를 저장하세요
- **예상 시간**: 10분

### 심화 과제 ⭐️⭐️

**과제 2**: 더블 클릭 감지하기

- **목표**: 버튼을 빠르게 두 번 누르면 "Double Click!" 출력
- **요구사항**:
  1. 0.3초 이내에 두 번 눌러야 함
  2. 일반 클릭과 구분되어야 함
- **힌트**: 마지막 클릭 시간을 저장하고 비교하세요
```
